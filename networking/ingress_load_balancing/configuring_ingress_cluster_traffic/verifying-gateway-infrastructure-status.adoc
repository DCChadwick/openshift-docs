= Verifying Gateway infrastructure status
include::_attributes/common-attributes.adoc[]
:context: verifying-gateway-infrastructure-status

toc::[]

[role="_abstract"]
The Gateway API uses status conditions to communicate the current state of resources. Understanding these conditions helps you verify that your Gateway infrastructure is properly configured and functioning.

When you deploy load balancing infrastructure using the Gateway API, you can review the status and conditions of your `GatewayClass` and `Gateway` resources. This allows you to confirm that the controller has accepted your configuration, that the underlying data plane is successfully programmed, and that there are no routing conflicts preventing traffic from flowing.

To verify that your Gateway infrastructure is functioning correctly, complete the following tasks:

* Understand `GatewayClass` status conditions to verify that the controller has claimed the class and that your installed API version is compatible.
* Review `Gateway` and listener status conditions to pinpoint data plane failures, configuration errors, or negative polarity conflicts.
* Query Gateway infrastructure status using the CLI to quickly validate your deployment and retrieve assigned IP addresses.
