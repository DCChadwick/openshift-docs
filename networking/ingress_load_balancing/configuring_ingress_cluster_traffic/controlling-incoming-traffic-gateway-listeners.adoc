= Controlling incoming traffic with Gateway listeners
include::_attributes/common-attributes.adoc[]
:context: controlling-incoming-traffic-gateway-listeners

toc::[]

[role="_abstract"]
Cluster administrators and network engineers need to expose applications to external networks securely. To ensure your applications are safely accessible, you must define exactly how your load balancer accepts incoming traffic. Controlling these entry points ensures that user requests are accurately and efficiently directed to the right services, while preventing unauthorized access and routing conflicts.

The Gateway API allows you to control this traffic flow using listeners. A listener defines the designated network port, protocol, and hostname for your Gateway. By configuring listeners, you can specify secure TLS connections, dictate how traffic is terminated, and restrict which application routes are permitted to attach to the Gateway.

To successfully manage your incoming traffic with Gateway listeners, complete the following tasks:

* Configure listener routing and security settings to define the ports, protocols, hostnames, and TLS certificates for your incoming traffic.
* Resolve routing conflicts between multiple listeners by applying conflict management rules to ensure overlapping hostnames or ports are routed correctly.
* Troubleshoot listener connections by monitoring listener status conditions to identify and resolve configuration errors.
