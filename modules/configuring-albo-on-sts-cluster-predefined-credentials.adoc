= Creating an IAM role for the AWS Load Balancer Controller

The `CredentialsRequest` object for the AWS Load Balancer Controller must be set with a manually provisioned IAM role.

.Procedure

. Download the `CredentialsRequest` custom resource (CR) for the **controller** and store it in a directory. Since you are installing version 1.1.0, use the `release-1.1` branch:
+
[source,terminal]
----
$ curl -o <path_to_directory>/controller-credentials-request.yaml \
  https://raw.githubusercontent.com/openshift/aws-load-balancer-operator/release-1.1/hack/controller/controller-credentials-request.yaml
----

. Use the `ccoctl` utility to create the AWS IAM role:
+
[source,terminal]
----
$ ccoctl aws create-iam-roles \
   --name <name> \
   --region=<aws_region> \
   --credentials-requests-dir=<path_to_directory> \
   --identity-provider-arn=arn:aws:iam::<aws_account_id>:oidc-provider/<oidc_issuer_url>
----

. Note the ARN for the AWS Load Balancer Controller role (e.g., `arn:aws:iam::777777777777:role/<name>-aws-load-balancer-operator-aws-load-balancer-controller`). You will need this ARN when creating the `AWSLoadBalancerController` custom resource instance.
