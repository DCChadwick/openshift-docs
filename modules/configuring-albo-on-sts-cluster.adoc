= Configuring the ARN role for the AWS Load Balancer Operator

You must configure the ARN role during the subscription creation for the Operator.

.Procedure

. Create the `aws-load-balancer-operator` project:
+
[source,terminal]
----
$ oc new-project aws-load-balancer-operator
----

. Create an `OperatorGroup` object:
+
[source,terminal]
----
$ cat <<EOF | oc apply -f -
apiVersion: operators.coreos.com/v1
kind: OperatorGroup
metadata:
  name: aws-load-balancer-operator
  namespace: aws-load-balancer-operator
spec:
  targetNamespaces: []
EOF
----

. Create a `Subscription` object. **Important:** You must include the `config.env` section with the ARN you created in the previous section.
+
[source,terminal]
----
$ cat <<EOF | oc apply -f -
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: aws-load-balancer-operator
  namespace: aws-load-balancer-operator
spec:
  channel: stable-v1
  name: aws-load-balancer-operator
  source: redhat-operators
  sourceNamespace: openshift-marketplace
  config:
    env:
    - name: ROLEARN
      value: "<albo_role_arn>" <1>
EOF
----
<1> Replace `<albo_role_arn>` with the ARN role you noted in the previous section (e.g., `arn:aws:iam::777777777777:role/<name>-aws-load-balancer-operator-aws-load-balancer-operator`).
